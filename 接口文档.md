# DIY Multi-Agents 接口文档

基础地址：http://127.0.0.1:8000/api/v1

## 创建推荐任务

POST /recommendations

请求：

~~~json
{"text": "预算四万到五万，主要玩 4K 3A 游戏，只要主机"}
~~~

响应状态码：202，返回 task_id 和 queued 状态。

## 查询任务状态

GET /recommendations/{task_id}/status

返回 status、progress、current_agent、degraded_reason、follow_up_questions 和 agent_runs。

任务状态：

- queued：等待运行。
- running：LangGraph 正在执行。
- completed：已发布通过硬校验的方案。
- needs_clarification：需要用户补充需求，应修改输入并创建新任务。
- degraded：没有可安全发布的方案或需求不可实现。
- failed：编排发生未处理异常。

每个任务固定返回六条 Agent 记录；未运行节点为 skipped。

## 获取最终结果

GET /recommendations/{task_id}

任务尚未完成或正在等待澄清时返回 409。最终数据包含 profile、parts、budget_check、checks、agent_runs、provenance 和 risks。降级任务的 parts 为空。

## 查询执行轨迹

GET /recommendations/{task_id}/trace

返回六节点的 status、input_summary、output_summary、latency_ms、iterations、tool_calls 和 error。

## 其他接口

- POST /requirements/parse：只执行确定性需求解析。
- POST /compatibility/check：执行插槽、内存、尺寸和电源检查。
- GET /health：健康检查。

## 错误

- 404：任务不存在，或服务重启后内存任务已丢失。
- 409：结果尚未准备好或任务正在等待补充信息。
- 422：请求字段校验失败。
